# Problem Statement

Management of user accounts

# Scope of the project

* Program can store as many accounts as user wish with seperate notepads for each account.
* However it does everything in single runtime session therefore stores data temporarily and lose the data once program ends.
* It allows any user to see all registered accounts without revealing sensitive data. 
* It does not mask password and security is quite primitive.

# Target users

A primitive software for storing banking info., logs, writing memos, all while enabling different users to practice the the same facility.

# High level features
* Enables signing as many accounts as user wish
* Notepad feature for every account.
* Availability of all registered account.
